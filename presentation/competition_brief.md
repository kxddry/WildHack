## Задача
Командный трек: спроектировать систему автоматического вызова транспорта на склады на основе прогноза отгрузок.

Включает:
1. Предсказательную модель на тренировочных данных.
2. Архитектуру и бизнес-логику сервиса автовызова транспорта (от прогноза → к заявкам на ТС).

## Требования к прототипу
- Веб-сервис (любой стек) с прогнозной моделью в основе.
- Логика работы с прогнозом: периодичность, горизонт планирования, преобразования данных.
- Подход к оценке качества: бизнес-метрики, тестирование, бейзлайн.
- Пути развития: масштабирование, доп. данные.
- GitHub-репозиторий с README, описанием бизнес-допущений, инструкциями запуска.

## Данные (parquet)
Колонки train:
- `route_id` — ID маршрута
- `office_from_id` — склад отправления (фиксирован)
- `timestamp` — момент времени
- `status_1..status_8` — кол-во товаров на этапах обработки за последние 30 минут
- `target_2h` — **target**: кол-во емкостей, отгруженных по маршруту за последние 2 часа

Test: без статусов и target. Нужно предсказать `target_2h` на каждую точку.

## Метрика
**WAPE + |Relative Bias|**

```python
class WapePlusRbias:
    def calculate(self, y_true, y_pred):
        wape = np.abs(y_pred - y_true).sum() / y_true.sum()
        rbias = np.abs(y_pred.sum() / y_true.sum() - 1)
        return wape + rbias
```
